# Contributing to BMT

Thank you for contributing to BMT! To maintain code quality, please adhere to these guidelines:

## Code Style & Standards
- All code must follow Clean Architecture, Domain-Driven Design (DDD), and SOLID principles.
- Use NestJS module conventions.
- Format files with Prettier and run `pnpm lint` before submitting PRs.

## Branching Strategy
- Create feature branches off the `develop` branch.
- Format: `feature/short-description` or `bugfix/issue-id`.

## Commit Messages
- Follow Semantic Commit format: `feat: add recommend deduplication` or `fix: check rollback checksums`.
