# Project Documentation Directory

This directory stores system requirements, architecture blueprints, database layouts, and API specifications.

## Documentation Index

1.  **Product Requirements (PRD):** Comprehensive product boundaries and workspace designs.
2.  **Software Specifications (SRS):** Detailed module requirements and technical rules.
3.  **Database Architecture:** Layered table layouts, sharding rules, and indexing strategies.
4.  **Database Standards:** Naming, primary/foreign keys, and RLS constraint rules.
5.  **Entity Relationship Design (ERD):** Structural relational tables and cascade dependencies map.
6.  **REST API Specifications:** HTTP route specifications and request validations.
7.  **UX Navigation Architecture:** Directory routes, workspaces routing, and security guards.
8.  **UI/UX Design Specifications:** Screen interfaces, component behaviors, and style variables.
9.  **Backend Architecture Blueprint:** Onion layer folder boundaries and queue workers setups.
10. **Frontend Architecture Blueprint:** Feature-driven component layouts and caching engines.
11. **Sprint Planning & Timeline:** Iteration goals and release milestones.
12. **Development Playbook:** Coding standards and verification workflows.
