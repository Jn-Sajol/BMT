# BMT — Semantic Versioning Policy

BMT strictly adheres to Semantic Versioning (SemVer 2.0.0).

---

## 1. Version Format
Versions are formatted as `MAJOR.MINOR.PATCH` with optional pre-release suffixes (e.g. `rc-1`).

*   **MAJOR:** Incremented for incompatible API contracts or breaking database schema changes.
*   **MINOR:** Incremented for new backward-compatible feature additions (e.g., adding a new AI provider to the registry).
*   **PATCH:** Incremented for backward-compatible bug fixes and security hotfixes.
