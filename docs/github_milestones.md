# BMT — GitHub Milestones Documentation

This document describes how BMT aligns feature rollouts and releases using milestones.

---

## 1. Milestones Layout
*   **`v1.0.0-rc-1` (Release Candidate 1):** Focuses on final architecture audits, performance baseline validations, and go-live preparation.
*   **`v1.0.0` (Production Release):** Deploying container configurations and Helm packaging templates to public clusters.
*   **`v1.1.0` (Post-Release optimization):** Dynamic capabilities loading and AI recommendation templates extensions.
