# BMT — GitHub Labels Documentation

This document lists standard GitHub label definitions used to categorize issues and pull requests.

---

## 1. Issue Classifications
*   `bug`: Indicates unexpected behaviors or system errors. (Color: `#d73a4a`)
*   `enhancement`: New feature proposals or improvements. (Color: `#a2eeef`)
*   `security`: Security vulnerability investigations. (Color: `#b60205`)

## 2. Process Classifications
*   `wip`: Work in progress. (Color: `#fbca04`)
*   `review-needed`: Requires architectural sign-off. (Color: `#0075ca`)
*   `approved`: Approved for merging. (Color: `#0e8a16`)
